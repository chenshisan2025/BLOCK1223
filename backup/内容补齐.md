    •    全站术语统一（Treasury/Referral/Burn/Pool）
    •    所有“演示数据/模拟合作/合规声明”都有明确文案位（避免误导）

⸻

1) Key 命名规范（完整版，强执行）

1.1 顶层命名空间
    •    brand：品牌名/口号/短句
    •    terms：全站术语词典（强制引用）
    •    nav：导航与路由入口
    •    seo：页面标题/描述/OG 文案
    •    common：通用按钮/状态/错误/表单/ARIA/法律声明
    •    auth：登录/注册/账号
    •    game：玩法参数与通用规则文案模板（可被多个页面复用）
    •    页面级：home rewards leaderboard invite transparency help sponsors investors
    •    legal：独立法律/规则页面（Terms/Privacy/Official Rules/AMOE 等）

说明：你们可以把 JSON 拆分成多文件，但 key 结构保持一致即可。

1.2 Key 风格
    •    camelCase（例如 titleLine1、verifyOnChain）
    •    列表项用数字键：items.0.title / faq.items.3.q（避免数组 returnObjects 引发维护问题）

1.3 叶子字段统一语义（必须按这个写）
    •    title subtitle body note hint desc
    •    cta.*：按钮
    •    label.*：标签/字段名
    •    badge.*：徽章
    •    status.*：状态
    •    empty.*：空态
    •    error.*：错误
    •    toast.*：轻提示
    •    tooltip.*：悬浮说明
    •    aria.*：无障碍/可读性

1.4 插值变量规范
    •    统一 {{variableName}}
    •    数字与规则参数尽量用插值（可由配置/后端注入）
    •    示例：{{reviveCostBwt}}、{{ticketCostBwt}}、{{reviveSeconds}}、{{drawIntervalHours}}、{{luckyWinnersCount}}、{{referralPct}}、{{burnPct}} 等

1.5 术语强约束（非常关键）

凡是出现以下词，必须从 terms.* 引用，禁止页面里再造同义词：
    •    terms.treasury（平台金库 / Treasury）
    •    terms.referral（邀请返佣 / Referral）
    •    terms.burn（销毁 / Burn）
    •    terms.pool（奖池 / Pool）
    •    terms.bwtToken（BWT（Token 代币） / BWT (Token)）

1.6 演示合规与“非误导”约束

涉及数据展示、Sponsor 注入演示、交易流等：
    •    必须有 common.badge.demo 或 common.label.simulated
    •    必须有 common.legal.simulatedDisclaimer / common.legal.simulatedPartnership 文案位
