下面给你一套可直接喂给前端的 i18n 方案：包含
    1.    Key 命名规范（全站统一口径、便于组件化与长期维护）
    2.    全站 i18n JSON（zh / en）（覆盖：Home / Rewards / Leaderboard / Invite / Transparency / Help Center 全文 / Sponsors 简版 / Investors 简版 + 全站通用）

你可以直接把下面两份 JSON 分别保存为：
    •    src/locales/zh.json
    •    src/locales/en.json

并在 react-i18next 初始化时加载。

⸻

1) Key 命名规范（强烈建议按此执行）

1.1 命名结构
    •    **一级命名空间（Namespace）**按“域”划分：
    •    brand（品牌口号/一句话）
    •    terms（术语词典：Treasury/Referral/Burn/Pool 等）
    •    nav（导航/菜单）
    •    common（通用按钮、标签、状态、空态、Toast、免责声明）
    •    页面级：home / rewards / leaderboard / invite / transparency / help / sponsors / investors / footer

1.2 叶子 key 统一用这些词（避免同义混乱）
    •    title / subtitle / desc / body / note
    •    cta.*（按钮）
    •    label.*（短标签）
    •    badge.*（徽章）
    •    status.*（状态文案）
    •    empty.*（空态）
    •    toast.*（提示）
    •    table.*（表头/列表字段）
    •    tabs.*（Tab）

1.3 列表/要点的写法
    •    不用数组（避免 returnObjects: true），用数字下标当对象键：
    •    home.aha.points.0 / home.aha.points.1
    •    home.faq.items.0.q / home.faq.items.0.a

1.4 插值与复数
    •    动态值统一用 {{ }}：
    •    {{amount}} / {{count}} / {{time}} / {{rank}} / {{score}} / {{reward}}
    •    英文复数用 i18next 复数规则：
    •    time.minutesAgo_one / time.minutesAgo_other

1.5 术语强约束（必须引用 terms）
    •    全站禁止出现 Dev / Ref 这类缩写与混用口径
    •    统一用：
    •    terms.treasury（平台金库 / Treasury）
    •    terms.referral（邀请返佣 / Referral）
    •    terms.burn（销毁 / Burn）
    •    terms.pool（奖池 / Pool）
